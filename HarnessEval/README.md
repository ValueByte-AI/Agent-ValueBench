# HarnessEval 🧪

HarnessEval runs Agent-ValueBench through alternative agent harnesses beyond the main OpenAI-compatible function-calling runner. It is used to study whether the execution harness itself changes an agent's value preferences.

This module supports two families of experiments:

- **Standard harness evaluation**: run the selected Agent-ValueBench cases with OpenClaw, Codex, or Claude Code without any value-priority skill.
- **Skill-injection harness evaluation**: install a model-specific value-priority skill into the task environment, explicitly tell the agent to use it, and rerun the selected cases.

The generated trajectories can then be judged by [`ValueEval`](../ValueEval/README.md) using the same rubric-based judging and value-priority pipeline used for the main trajectories.

Any score or verifier result shown inside Harbor or OpenClaw runs is only a quick health check for whether the agent completed the task and whether the harness is working normally. Those harness-side scores are not the official Agent-ValueBench value scores. Official value adherence and value priority results should always be computed from the exported trajectories with [`ValueEval`](../ValueEval/README.md).

## Setup 🚀

### 1. Install Dependencies

```bash
cd HarnessEval
uv sync
```

Clone and install Harbor from upstream:

```bash
git clone https://github.com/harbor-framework/harbor.git harbor
git -C harbor checkout 0533a59c41ce435d9e59ff8c82da67f6e5b6edc7
uv tool install ./harbor
```

The commands below assume the `harbor` executable is available after `uv tool install ./harbor`.

### 2. Pre-Pull the Docker Base Image

Each Harbor task builds a small per-case Docker environment from the same Python base image:

```dockerfile
FROM python:3.12-alpine
```

Pull it once before launching a large concurrent run. This prevents many cases from trying to resolve and download the same base image at the same time.

```bash
docker pull python:3.12-alpine
```

This step is especially useful before running Harbor with high `--n-concurrent` values.

### 3. Configure API Access

OpenClaw scripts read API settings from environment variables. You can either create `.env` from the template:

```bash
cp .env.example .env
```

or pass variables inline before each command:

```bash
OPENAI_API_BASE=<api_url>
OPENAI_API_KEY=<api_key>
MODEL_ID=<model_id>
```

For Harbor agents, pass credentials with `--ae KEY=VALUE`. The required keys depend on the agent:

| Harness | Required credential style |
| --- | --- |
| OpenClaw | `OPENAI_API_BASE`, `OPENAI_API_KEY`, `MODEL_ID` |
| Harbor Codex | `OPENAI_API_KEY`; optionally `OPENAI_BASE_URL` for non-default endpoints |
| Harbor Claude Code | `ANTHROPIC_AUTH_TOKEN` or `ANTHROPIC_API_KEY`; optionally `ANTHROPIC_BASE_URL` |

Use the exact model id expected by your endpoint, for example `<model_id>`.

### 4. Prepare Harness Assets

HarnessEval derives its local assets from the canonical dataset at the repository root. Run this once before generating Harbor tasks or launching OpenClaw:

```bash
uv run python scripts/prepare_harness_assets.py
```

This creates:

```text
cases/       # 111 selected cases
environment/ # Environment files for harness runs
```

The case subset contains all `mft08`, `vsm13`, and `nfcc2000` cases, plus the `pvq40` cases whose value pair is drawn only from `Achievement`, `Benevolence`, `Conformity`, `Hedonism`, `Power`, `Security`, `Self-Direction`, `Stimulation`, `Tradition`, and `Universalism`.

The generated `environment/` directory copies the root `environment/` files and removes only root-level `anyOf`, `oneOf`, and `allOf` fields from each tool `parameters` schema. This matches the tool-schema cleanup used by the main trajectory runner for the model families evaluated here.

### 5. Choose Cases

To run all prepared cases with OpenClaw, create a case-id list:

```bash
find cases -maxdepth 1 -name '*.json' -printf '%f\n' \
  | sed 's/\.json$//' \
  | sort > target_case_ids.txt
```

For skill experiments, the relevant case-id files are already provided:

```text
mft08_skill_case_ids.txt
pvq40_10dim_skill_case_ids.txt
```

### 6. Generate Harbor Tasks

Codex and Claude Code run through Harbor task directories. Generate those directories once before launching Harbor-based runs:

```bash
uv run python adapters/agent_valuebench/run_adapter.py \
  --output-dir datasets/agent_valuebench
```

OpenClaw does not use these Harbor task directories; it reads directly from `cases/` and `environment/`.

## Standard Harness Runs 🧭

### A. Run OpenClaw Without Skill

```bash
OPENAI_API_BASE=<api_url> \
OPENAI_API_KEY=<api_key> \
MODEL_ID=<model_id> \
uv run python scripts/run_openclaw_batch.py \
  --case-ids-file target_case_ids.txt \
  --job-name openclaw-<model_name> \
  --results-dir results/openclaw \
  --n-concurrent 4 \
  --timeout 600
```

Optional export to the same flat trajectory layout used by `ValueEval`:

```bash
uv run python scripts/export_skill_run_outputs.py \
  --kind openclaw \
  --input-dir results/openclaw/openclaw-<model_name> \
  --output-dir ../result_harness/<model_name>/openclaw
```

### B. Run Harbor Codex Without Skill

```bash
harbor run --path datasets/agent_valuebench \
  --agent codex \
  --model <model_id> \
  --ae OPENAI_API_KEY=<api_key> \
  --ae OPENAI_BASE_URL=<api_url> \
  --n-concurrent 4
```

Optional export:

```bash
uv run python scripts/export_skill_run_outputs.py \
  --kind harbor \
  --input-dir jobs/<harbor_job_name> \
  --output-dir ../result_harness/<model_name>/codex
```

### C. Run Harbor Claude Code Without Skill

```bash
harbor run --path datasets/agent_valuebench \
  --agent claude-code \
  --model <model_id> \
  --ae ANTHROPIC_AUTH_TOKEN=<api_key> \
  --ae ANTHROPIC_BASE_URL=<api_url> \
  --n-concurrent 8
```

Optional export:

```bash
uv run python scripts/export_skill_run_outputs.py \
  --kind harbor \
  --input-dir jobs/<harbor_job_name> \
  --output-dir ../result_harness/<model_name>/claudecode
```

## Skill-Injection Harness Runs 🎯

Skill experiments use model-specific skills that encode a fixed value-priority order. The preparation scripts do two things:

1. generate a Harbor dataset containing only the target value-system cases;
2. mount the selected skill into each task and add an explicit instruction telling the agent to read `SKILL.md` and `references/conflict-examples.md`.

The checked-in skill sources and preparation scripts currently support only these model-specific skill variants:

| Model name | Skill slug |
| --- | --- |
| `claude-sonnet-4.6` | `claude_sonnet_4_6` |
| `gpt-5.4` | `gpt_5_4` |
| `kimi-k2.5` | `kimi_k2_5` |

Use the matching `<model_slug>` in the commands below. Running skill-injection experiments for another model requires adding a new model-specific skill source and updating the preparation scripts; otherwise use the standard harness commands without skill injection.

### A. Prepare MFT08 Skill Datasets

```bash
uv run python prepare_mft08_skill_experiment.py
```

This creates:

```text

datasets/agent_valuebench_mft08_skill_gpt_5_4
datasets/agent_valuebench_mft08_skill_kimi_k2_5
```

The matching skill roots are:


## Argument Reference ⚙️

### `adapters/agent_valuebench/run_adapter.py`

| Argument | Default | Description |
| --- | --- | --- |
| `--output-dir` | `datasets/agent_valuebench` | Directory where Harbor task directories are written. |
| `--case-dir` | `cases/` | Source case JSON directory. |
| `--env-dir` | `environment/` | Source environment directory. |
| `--case-ids` | none | Generate only the listed case ids. |
| `--limit` | none | Generate only the first N cases after sorting. |

### `scripts/prepare_harness_assets.py`

| Argument | Default | Description |
| --- | --- | --- |
| `--case-source-dir` | `../case` | Canonical root case directory. |
| `--environment-source-dir` | `../environment` | Canonical root environment directory. |
| `--case-output-dir` | `cases/` | HarnessEval case subset output directory. |
| `--environment-output-dir` | `environment/` | HarnessEval environment output directory. |

### OpenClaw Batch Scripts

Applies to `scripts/run_openclaw_batch.py`, `scripts/run_openclaw_mft08_skill_batch.py`, and `scripts/run_openclaw_pvq40_skill_batch.py`.

| Argument | Required | Default | Description |
| --- | --- | --- | --- |
| `--case-ids-file` | one of `--case-ids-file` or `--case-ids` | none | Text file containing one case id per line. |
| `--case-ids` | one of `--case-ids-file` or `--case-ids` | none | Case ids passed directly on the command line. |
| `--skills-dir` | only for skill scripts | none | Directory containing the skill folder to mount into the OpenClaw container. |
| `--timeout` | no | `600` | Per-case wall-clock timeout in seconds. |
| `--n-concurrent` | no | `4` | Number of cases to run in parallel. |
| `--results-dir` | no | `results/openclaw`, `results/openclaw_skill_mft08`, or `results/openclaw_skill_pvq40` | Parent directory for OpenClaw outputs. |
| `--job-name` | no | script-specific | Subdirectory name under `--results-dir`. |

OpenClaw environment variables:

| Variable | Required | Description |
| --- | --- | --- |
| `OPENAI_API_BASE` | yes | Base URL used by OpenClaw's configured provider. |
| `OPENAI_API_KEY` | yes | API key for the model endpoint. |
| `MODEL_ID` | yes for batch scripts | Model id passed to the endpoint. |
| `OPENCLAW_AGENT_TIMEOUT_SECONDS` | no | Optional internal OpenClaw agent timeout override. |
| `OPENCLAW_LLM_IDLE_TIMEOUT_SECONDS` | no | Optional internal OpenClaw LLM idle timeout override. |

### `scripts/export_skill_run_outputs.py`

| Argument | Required | Description |
| --- | --- | --- |
| `--kind` | yes | `harbor` for Harbor job outputs, `openclaw` for OpenClaw result directories. |
| `--input-dir` | yes | Job/result directory to read. |
| `--output-dir` | yes | Flat trajectory output directory. |

### Harbor `run` Flags Used Here

HarnessEval uses Harbor's own CLI for Codex and Claude Code. The relevant flags are:

| Flag | Description |
| --- | --- |
| `--path` | Harbor task dataset directory, such as `datasets/agent_valuebench`. |
| `--agent` | Agent backend, such as `codex` or `claude-code`. |
| `--model` | Model id passed to the selected agent. |
| `--ae KEY=VALUE` | Environment variable passed to the agent process. Use it for API keys, base URLs, and provider-specific settings. |
| `--n-concurrent` | Number of concurrent tasks. |
| `--job-name` | Optional explicit Harbor job name. |
| `--jobs-dir` | Optional parent directory for Harbor job outputs. |

Run `harbor run --help` for the full Harbor CLI reference.

## Notes ✅

- Do not commit `.env`, API keys, generated `datasets/`, `jobs*/`, or `results/`.
- The bundled skills are model-specific. Use the skill directory whose slug matches the model being tested.
- `prepare_mft08_skill_experiment.py` and `prepare_pvq40_10dim_skill_experiment.py` may write local manifest files for auditing; those files are not required for running the experiment.
- For final value scoring, run the exported trajectories through `ValueEval`.
